# Bug Triage Policy (Sprint 6)

## Severity
- P1: sistem unusable/data loss/security leak. Hotfix immediate.
- P2: fitur utama terganggu, ada workaround terbatas.
- P3: minor UX/performance, masuk backlog sprint berikutnya.

## SLA
- P1: acknowledge < 15 menit, mitigasi < 2 jam.
- P2: acknowledge < 2 jam, fix < 24 jam.
- P3: prioritas planning sprint.

## Workflow
1. Reproduce issue.
2. Label severity + owner.
3. Patch + regression check.
4. Verify with reporter.
5. Close with root cause note.
